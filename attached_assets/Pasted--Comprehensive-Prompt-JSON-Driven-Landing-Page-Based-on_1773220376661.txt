# Comprehensive Prompt: JSON-Driven Landing Page (Based on Panoramic Exports)

> Use this prompt to build a **complete production-ready landing page** for any B2B manufacturer/exporter business. All content is driven from a single JSON data file — change the JSON, and the entire site updates automatically (like slugs in Next.js).

---

## 🎯 High-Level Goal

Build a **single-page React landing page** with:
- **Vite + React 18 + TypeScript** as the framework
- **Tailwind CSS** for styling (with CSS variable-based theming)
- **Framer Motion** for scroll-reveal animations
- **A centralized JSON data file** (`siteData.json` or `data/siteContent.ts`) that drives ALL text, images, colors, links, SEO meta, form config, and contact info — so changing the JSON instantly reflects on the UI
- **Firebase Firestore** for form lead storage
- **Nodemailer SMTP** for sending inquiry + thank-you emails
- **Express server** for the API layer (integrated via Vite dev middleware)
- **Vercel + Netlify** deployment support

## 📐 Component-by-Component Build Specification

### 1. [Header.tsx](file:///home/rachit/build/CW/CW-PELanding-main/client/components/Header.tsx)
**Data source:** `siteContent.brand`, `siteContent.navigation`, `siteContent.theme`

| Feature | Detail |
|---|---|
| Top banner bar | Gold/accent color bar with `brand.tagline` text (hidden on mobile) |
| Logo | `brand.logoSrc` as `<img>`, links to `/` |
| Nav items | Rendered from `navigation[]` array. Supports both route navigation AND hash-scroll (e.g. `/#about`) |
| Mobile menu | Hamburger toggle with smooth `max-h` transition. Uses `lucide-react` Menu/X icons |
| Scroll behavior | [handleScrollNav()](file:///home/rachit/build/CW/CW-PELanding-main/client/components/Header.tsx#16-39) function: if already on homepage, smooth-scrolls to `#id`; otherwise navigates to `/` first, then scrolls after 200ms |
| Hover effect | Underline slides in from left on hover (CSS transition `w-0 → w-full`) |

### 2. `CarouselBags.tsx` (Hero Carousel)
**Data source:** `siteContent.hero.carouselImages`

| Feature | Detail |
|---|---|
| Desktop | Framer Motion–animated horizontal slide (`animate: x: -current*100%`). Left/right chevron arrows + clickable dot indicators. Auto-advances every 3 seconds |
| Mobile | Static stacked images (`flex-col`), no carousel — all images visible |
| Auto-slide | `useEffect` + [setTimeout](file:///home/rachit/build/CW/CW-PELanding-main/client/components/CarasolBags.tsx#12-16) loop, resets on manual navigation |

### 3. [CatalogueForm.tsx](file:///home/rachit/build/CW/CW-PELanding-main/client/components/CatalogueForm.tsx)  ⭐ MOST CRITICAL
**Data source:** `siteContent.form`, `siteContent.email`

| Feature | Detail |
|---|---|
| Layout | White card, rounded-2xl, shadow-2xl. Max-width [md](file:///home/rachit/build/CW/CW-PELanding-main/AGENTS.md), centered |
| Badge | Top-center pill badge (e.g. "DIRECT FROM FACTORY") |
| Heading | "Get Your **Catalog** Now!" (highlight word in red) |
| Fields | Dynamically rendered from `form.fields[]` — each has `name`, `label`, `type`, `placeholder`, `required`, `rows` (for textarea) |
| Submit flow | **Step 1:** Save to Firebase Firestore (`addDoc` to `form.firestoreCollection`) with `fullName`, `location`, `email`, `message`, `source`, `createdAt` (serverTimestamp) |
| | **Step 2:** Fire-and-forget `fetch('/api/send-email')` — sends owner notification + customer thank-you email. Errors logged but don't block UI |
| | **Step 3:** Show success checkmark + title + message |
| Reset | After `form.resetDelayMs` (8 sec), clears form and shows input again |
| States | `isSubmitting` (spinner + disabled), `isSubmitted` (success view), `error` (red error box) |
| Privacy note | Small gray text below submit button |

## 🔧 API & Backend

### `/api/send-email` (POST)
**Data source:** `siteContent.email`

| Aspect | Detail |
|---|---|
| Framework | Vercel serverless function (also wrapped by Express for dev) |
| Transport | Nodemailer SMTP (host, port, user, pass from [.env](file:///home/rachit/build/CW/CW-PELanding-main/.env)) |
| Two emails sent | **1) Owner notification** — branded HTML email with customer details table + reply-to customer's email. **2) Customer thank-you** — branded HTML confirmation email |
| Email HTML | Styled HTML with dark-mode `@media (prefers-color-scheme: dark)` support. Logo header, data box, footer bar. All brand text from JSON |
| Validation | Requires `fullName`, `email`, `location`. Returns 400 if missing |
| Error handling | 500 with error details if email fails |

### Express Server ([server/index.ts](file:///home/rachit/build/CW/CW-PELanding-main/server/index.ts))
- `POST /api/send-email` → calls [SendEmailHandler](file:///home/rachit/build/CW/CW-PELanding-main/api/send-email.ts#4-206) (adapted from Vercel types)
- `GET /api/ping` → returns `PING_MESSAGE` from env
- `GET /api/demo` → returns demo response
- Middleware: CORS, JSON body parser, URL-encoded body parser

### Production Server ([server/node-build.ts](file:///home/rachit/build/CW/CW-PELanding-main/server/node-build.ts))
- Serves built SPA from `dist/spa/`
- React Router catch-all: serves [index.html](file:///home/rachit/build/CW/CW-PELanding-main/index.html) for all non-API routes

## 🔁 How the JSON-Driven Approach Works

1. **All components import from `siteContent`** — e.g. `import { siteContent } from '@/data/siteContent'`
2. **No hardcoded strings in components** — every heading, paragraph, image path, link, and array of items comes from the data file
3. **To create a new site:** duplicate the project, replace `siteContent` with new brand data, swap images in `/public/`, update `.env` with new Firebase/SMTP credentials
4. **Section IDs** like `about`, `ourproduct`, `contact`, `faq` are defined in the JSON, so navigation targets are also data-driven
5. **Icons** are referenced by string name (e.g. `"CheckCircle2"`) and resolved at runtime via a `lucide-react` icon map
6. **SEO meta** in `index.html` should also be templated from the JSON (or use a build-time script to inject them)
